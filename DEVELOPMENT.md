# Development Guide

## Prerequisites

- Node.js >= 20 (recommended: 20 LTS or later)
- npm

## Setup

```bash
npm install
```

This also runs `postinstall` which patches the scaffold's update notifier to avoid ETIMEDOUT errors in offline/CI environments.

## Commands

| Command | Purpose | Exit Code |
|---------|---------|-----------|
| `npm run build` | Build plugin + type check | 0 |
| `npm run test:unit` | Run unit tests (mocha) | 0 |
| `npx tsc --noEmit` | TypeScript type check only | 0 |
| `npm run lint:check` | Check formatting (prettier) + lint (eslint) | varies |
| `npm run start` | Dev mode (zotero-plugin serve) | - |

## Build Output

- `.scaffold/build/addon/` — plugin directory
- `.scaffold/build/zotero-theme-switcher.xpi` — installable XPI package

## Testing

### Unit Tests (no Zotero required)

```bash
npm run test:unit
```

Runs mocha tests against pure modules (JsonUtils, themes, Disposable). No Zotero installation needed.

### Integration Tests (requires Zotero)

```bash
npm run test
```

Uses `zotero-plugin test` which launches a Zotero instance. Requires Zotero to be installed locally.

## Manual Installation

1. Build: `npm run build`
2. Open Zotero → Tools → Plugins → Gear icon → Install from file
3. Select `.scaffold/build/zotero-theme-switcher.xpi`
4. Restart Zotero if prompted
5. Open any HTML page in the Reader — a theme button appears in the toolbar

## Known Issues

### Build exit code 1 (scaffold ETIMEDOUT)

**Status**: Fixed via `cross-env NO_UPDATE_NOTIFIER=1` in the build script.

The scaffold's `tiny-update-notifier` dependency makes an HTTP request to npm on every build. In offline/CI environments, this causes ETIMEDOUT which becomes an uncaught exception and triggers `process.exit(1)`, even though the build itself succeeds.

**Fix applied**: The build script sets `NO_UPDATE_NOTIFIER=1` which skips the network check entirely. A `postinstall` script patches the scaffold's CLI to respect this env var.

### ESLint flat config

ESLint 9 requires a flat config file. The project has `eslint.config.js` which imports `@zotero-plugin/eslint-config`. There are ~67 pre-existing code style issues (mostly `mocha/no-mocha-arrows` in test files).

### Zotero 9/10 Compatibility

**Status**: Unverified. The plugin declares `strict_max_version: "8.*"` in manifest.json.

The plugin uses `reader._iframeWindow` (private field) and `Zotero.Reader.registerEventListener` which may change in Zotero 9/10. No runtime testing has been performed against Zotero 9 or 10.

## Soft Copyright Materials

To regenerate the source code dump for software copyright applications:

```bash
node scripts/generate_source_dump.js
```

This generates `source_code_dump.txt` from all `src/**/*.ts` files. The output is gitignored.

## Regenerating Typed Definitions

The scaffold auto-generates `typings/prefs.d.ts` and `typings/i10n.d.ts` during build. These are tracked in git but can be regenerated by running `npm run build`.
